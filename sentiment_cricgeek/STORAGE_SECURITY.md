# CricGeek Database Storage

## Final Storage Choice

CricGeek uses SQLAlchemy with PostgreSQL or MySQL as the production-ready database layer. SQLite remains available only for lightweight local development through the default `sqlite:///./cricgeek_dev.db` setting.

Stored data includes:

- User accounts and bcrypt password hashes
- Hashed email verification and password reset tokens
- Refresh-token revocation records
- Writer profiles, topics, style, reputation, and aggregate scores
- Scored writing submissions and model component output
- Communities, memberships, and per-user rate limits

## Security Rules

- Production must set `DATABASE_URL` to PostgreSQL or MySQL. The app refuses SQLite when `ENVIRONMENT=production`.
- Prefer a managed PostgreSQL or MySQL service for production deployments so backups, patching, and encryption controls stay outside the app runtime.
- Production must set `JWT_SECRET_KEY` to at least 32 characters.
- Production must set `ENCRYPTION_KEY` to a Fernet key generated by `cryptography.fernet.Fernet.generate_key()`.
- Verification and password reset tokens are stored as SHA-256 hashes, not plaintext.
- Passwords are stored only as bcrypt hashes. Tune `BCRYPT_ROUNDS` if deployment hardware changes.
- Docker Compose now requires explicit `DB_PASSWORD`, `JWT_SECRET_KEY`, and `ENCRYPTION_KEY` values.

## Local Setup

Copy `.env.example` to `.env`, replace all placeholder secrets, then run:

```bash
docker compose up
```

or, for a local Python run:

```bash
python database.py init
uvicorn main_api:app --reload --host 0.0.0.0 --port 8000
```

## Writer Data Flow

When `/api/score` stores a scoring result, `WriterDataService` also updates the writer profile in the same transaction:

- `total_submissions`
- rolling `avg_eqs_score` and `avg_bqs_score`
- writing style from scoring components when available
- community-derived topic affinity
- reputation points
- metadata for the latest scored submission

## Final Data Map

All persisted product data should live in PostgreSQL or MySQL:

- Users and authentication records
- Writer profiles and reputation data
- Communities and membership records
- Scoring history and EQS/BQS outputs
- Qwen 3.5 quota tracking and rate-limit status
- Verification and password-reset token hashes
